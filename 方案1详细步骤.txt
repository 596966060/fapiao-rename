【Replit 删除重建 - 完整步骤指南】

================================
第1步：打开Replit并找到项目
================================

1. 用浏览器打开：https://replit.com
   (如果已登录，会看到你的项目列表)

2. 找到你的 invoice 项目
   (应该在最左边的项目栏中，名称可能是 "invoice" 或 "invoice-tool")

3. 找到项目右上角的三个点菜单 (⋯) 或项目卡片上的菜单
   点击它

================================
第2步：进入项目设置并删除
================================

4. 在弹出的菜单中找 "Settings"（设置）
   点击它

5. 页面会跳转到 Settings 页面
   向下滚动（往下翻）

6. 找到红色的 "Delete" 或 "Delete Repl" 按钮
   (通常在最下方)
   点击它

7. 会弹出确认框，要求你输入项目名称
   仔细输入你的项目名（比如 "invoice-tool" 或 "invoice"）
   然后点 "Delete" 按钮确认

8. 项目会被删除，页面返回主页面

================================
第3步：创建新项目并从GitHub导入
================================

9. 回到 Replit 主页
   点击左上角的 "+ Create" 按钮（蓝色的新建按钮）

10. 选择 "Import from GitHub"
    (如果没看到这个选项，点 "Templates" 再找)

11. 会跳转到 GitHub 授权页面
    点击 "Connect GitHub" 或 "Authorize Replit"
    按提示完成授权（可能需要输入GitHub密码）

12. 授权完成后，回到Replit
    在搜索框中输入你的仓库名：
    fapiao-rename
    或完整的GitHub地址：
    https://github.com/596966060/fapiao-rename

13. 找到 "fapiao-rename" 仓库
    点击它

14. Replit会自动填入项目配置
    你可以改项目名为 "invoice-tool"（可选）
    然后点击 "Import" 按钮

================================
第4步：等待导入完成
================================

15. Replit 开始下载和导入所有文件
    页面会显示进度
    等待一分钟左右完成

16. 导入完成后，会自动打开新项目
    左边是文件列表，中间是代码编辑器
    右边是 Shell 和 Run 按钮

================================
第5步：检查文件是否完整
================================

17. 在左边的文件列表中，检查是否有：
    ✓ app.py (主文件，应该是481行左右)
    ✓ requirements.txt
    ✓ templates 文件夹 (里面有 index.html)
    ✓ Procfile
    ✓ .gitignore

18. 特别检查 app.py 文件：
    在中间的代码编辑器中打开 app.py
    向下翻，找到 "_extract_fields" 方法
    如果能看到这个方法（里面有金额提取的逻辑），说明文件正确

================================
第6步：启动应用
================================

19. 点击右上角的 "Run" 按钮（绿色的）

20. 应用开始启动
    右下角会显示输出日志
    等待看到类似这样的信息：

    "✅ OCR 模型已准备好"
    或
    "正在初始化 OCR 模型..."

    这需要1-2分钟，因为OCR模型很大

21. 看到 "✅ OCR 模型已准备好" 后
    应用已准备好！
    右边会显示 "Webview" 区域，有个网址（看起来像 https://xxx.replit.dev）

================================
第7步：测试应用
================================

22. 点击右边显示的网址
    或在新标签页中打开这个网址
    会看到网页界面（紫色渐变背景，有"发票批量重命名工具"标题）

23. 上传你的发票图片测试
    用鼠标拖拽或点击上传区
    选择 screenshot-20260511-160543.png

24. 等待几秒钟
    应该看到识别结果：

    ✅ 日期: 2026-05-07
    ✅ 发票号: 26324000000154072606
    ✅ 购买方: 恭亲科技(上海)有限公司
    ✅ 销售方: 武进区潢堪饶殇水果店
    ✅ 金额: 6.31 元

    而不是空白或0000-01-01

================================
如果遇到问题
================================

问题1：导入时找不到仓库
→ 确认GitHub账号是否已连接
→ 尝试用完整URL：https://github.com/596966060/fapiao-rename

问题2：Run后提示错误
→ 检查 requirements.txt 是否存在
→ 在右边 Shell 中执行：pip install -r requirements.txt

问题3：OCR一直在初始化
→ 这很正常，第一次需要1-2分钟
→ 不要关闭浏览器或按停止按钮

问题4：识别结果还是空白
→ 在 Shell 中运行：python diagnose_replit.py
→ 把输出结果告诉我
